### Способ установки
- **Инфраструктура:** Yandex Cloud, развёрнута через Terraform
- **Установка Kubernetes:** kubeadm
- **CRI:** containerd (версия 1.7.27)

- **Версия Kubernetes:** v1.28.15

### Файлы Terraform
- `main.tf` — провайдер Yandex Cloud
- `variables.tf` — переменные
- `terraform.tfvars` — значения переменных (не коммитится в git)
- `network.tf` — VPC, подсеть, security group
- `vms.tf` — виртуальные машины
- `outputs.tf` — выходные значения
- `cloud-init/master.yaml` — настройка master-ноды
- `cloud-init/worker.yaml` — настройка worker-нод

### Пояснения

1. **Выбор CRI containerd:** containerd — это легковесный и производительный container runtime, рекомендованный Kubernetes. В отличие от Docker, он напрямую интегрирован с kubelet через CRI (Container Runtime Interface), что уменьшает накладные расходы.

2. **Запуск etcd на мастере:** etcd — это распределённое key-value хранилище, которое Kubernetes использует для хранения состояния кластера. В данной конфигурации etcd запущен как статический под на master-ноде (видно в выводе `kubectl get pods -n kube-system`). Для production-кластеров с высокой доступностью рекомендуется использовать выделенные ноды для etcd или stacked etcd на нескольких master-нодах.

3. **CNI Calico:** выбран Calico как популярное и надёжное решение для сетевой модели Kubernetes. Обеспечивает поддержку NetworkPolicy и высокую производительность.

4. **Развёртывание через Terraform:** инфраструктура описана как код (IaC), что позволяет легко воспроизводить окружение и управлять ресурсами.
